★ 핵심 · 출격 팝업 제거

별 조건 → 편성 화면 상시 통합

출격 확인 팝업의 유일한 신규 정보 = 별 획득 조건. 이걸 우측 스테이지 목표 스트립(CTA 바로 위)에 상시 노출 → 팝업 불요. 전투 시작 = 1탭 직행(확인/조정 모델의 반복 그라인딩 마찰 최소 보존). 상단 스테이지 헤더에도 ★1/3 요약.

★ 목표 = 덱 빌드에 영향

편성 중 목표를 보고 전략

별 조건(예: "60초 내" vs "영웅 HP 50%↑")이 덱 구성 전략을 바꿈 → 커밋 직전이 아니라 편성 중에 보여야 정답. 상시 노출 = 팝업보다 더 나은 정보 배치.

영웅 = 로스터 맨 앞

보유 유닛에 영웅 포함·고정 앞

로스터 첫 자리 = 영웅 그룹(골드 프레임·큰 카드). 클릭 → 0번 영웅 슬롯·다른 영웅 클릭 = 교체. 3상태 = 편성됨(✓) / 교체 가능 / 미해금(🔒 + 해금처). 일반 34는 잠금 없음(spec §376).

유닛 정보

롱클릭 → 모달 다이얼로그

전체화면 아님. 일러스트/idle + Lv · 스탯(HP·ATK 물/마·코스트·사거리) · 역할 · 태그 · 스킬 요약. 발견성(롱클릭=숨은 제스처·paranoia#3) = 온보딩 힌트로 안내.

영웅 슬롯 유지

0번 = 영웅 전용·사망=패배

영웅은 로스터에서 고르지만 덱에선 0번 고정 슬롯(spec §120). 어느 유닛이 영웅인지 시각 구분 필수(영웅 사망 = 패배). 골드 프레임 + "영웅" 배지.

계승 (v2.2)

예산 덱 · 테마 아이콘 · 전체 해금

코스트 예산(합 ≤ maxCost 70·spec §186) · 테마 아이콘 5종(dot 대체) · 일반 34 전체 해금 · Pretendard 단일. 태그 표기 = 별도 논의(MVP=텍스트).

전투 시작 (단순화)

OnClickBattleStart → 전투 직행

출격 팝업 없음 = `BattlePrepareManager.OnClickBattleStart`(:99) 가 바로 전투 씬 진입(spec §253 집계 그릇 그대로). 팝업 화면/매니저 불요 = 구현 단순.

스테이지 목표 UI

StageData → 별 3조건 표시

편성 우측 목표 스트립 = 스테이지 3별 목표(클리어 / 영웅HP≥X% / 시간≤T·spec §6.B⑥) + 기획득 별 read. plat = StageData 목표 필드 → 편성 목표 위젯 + 상단 ★n/3.

로스터 = 영웅+일반

UnitListManager 앞에 영웅 그룹

로스터 맨 앞 = 해금 영웅(선택/교체) + 미해금 영웅(🔒·해금처). 클릭 → DeckSlotManager 0번 배치(영웅 전용). 일반 = 예산 슬롯. 해금 상태 = §6.B② 게이트.

UnitInformationTootipPanel
