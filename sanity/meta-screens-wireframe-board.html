강화 설계 포인트

  • 로스터 관리의 핵심 화면 — 편성(전투용 라인업)과 분리된 영구 성장 진입.
  • 증가분 미리보기 현재값 → +Δ 명시. "얼마 올리는지" 즉시.
  • 스탯/스킬 탭 분리. 비용 = 재화 상단 상시.
  • 2축 방어 노출 방어/마법저항 둘 다(전투 위협 응수와 연결).
  • 일괄 강화 / 자동 방치형 정합 = "한 번에 N레벨" 편의 필요한지 = 검토.

환생 설계 포인트

  • 두 선택 위계 레벨 상승(고위험·고보상) = 강조 / 유지(안전) = 보조.
  • 테마 선택 = 환생의 변주 5테마가 다음 회차 색·적 성격. 진행감의 핵.
  • 리셋 경고 명확 무엇이 리셋(스테이지)·무엇이 유지(강화·도감)인지.
  • 게이트 표시 최소 10스테이지 — 미달 시 비활성 + 안내.

미션 설계 포인트 · 제안

  • MVP 포함? 업적이 이미 있어 중복 우려 — 일일 1~3개 가벼운 반복으로 시작 vs 출시 후로 이연.
  • 업적과 차별 미션 = 반복·리셋(일일/주간) / 업적 = 영구 1회. 둘 다 있으면 역할 분리 명확히.
  • 리셋 타이머 일일 복귀 동기 = 방치형 리텐션 핵심.
  • 보상 재화(광고 모델상 무리 없는 소액) · 과보상 금지.

[제안] 광고 보상(무료·광고 수익 모델 정합) + 재화 IAP 패키지 + (부위 BM 스킨) · 추천/재화/스킨 탭

상점 설계 포인트 · 제안

  • MVP 포함? IAP 허용(전략 예외) + 광고 모델 → 상점은 의미 큼. 단 MVP = 광고 보상 + 광고 제거 + 소액 재화부터.
  • 광고 보상 = 무료 핵심 "광고 보고 재화" = 광고 수익 + 유저 가치 동시. 첫 탭에 노출.
  • 광고 제거 IAP 가장 흔한 캐주얼 결제 — 저마찰 1품.
  • 부위 BM 스킨 아트 파이프라인이 부위 스왑이라 스킨 BM 자연(흰 마스터 틴트·동작 재생성 0) = 출시 후 확장 축.
  • 비-P2W 가드 재화 = 성장 가속(편의)·전투력 직접 판매 X = ★1 리스크·정책 안전.
7 열린 결정 (사용자)
  • 미션 = MVP 포함 여부 (업적과 중복 우려 vs 일일 리텐션). 포함 시 일일 몇 개·보상 규모.
  • 상점 = MVP 범위 (광고 보상+광고 제거만 vs 재화 IAP·스킨까지). BM 깊이.
  • 결과 별점/기록 넣을지(완성 동기) vs 방치형 단순화로 생략.
  • 강화 일괄/자동 방치형 편의(한 번에 N레벨·MAX) 필요 여부.
