01

파운데이션 — 3계층 토큰

컴포넌트는 절대 원시값(hex/px)을 직접 참조하지 않는다. 오직 Semantic만. → 5테마 스킨·다크/라이트·리브랜딩이 "값 교체"만으로 가능해진다. (현 갭: 화면 USS에 raw값 ~200곳 → 시맨틱화 대상.)

Primitive원시값 · 의미 없음--gold-500: #FFD24D
--space-2: 8px
→
Semantic의도 · 테마 스위치 지점--color-cta: var(--gold-500)
--inset-card: var(--space-4)
→
Componentsemantic만 참조.btn-cta { background:
var(--color-cta) }

SSOT 파이프(계승·Figma 불요): design-tokens.json → DesignTokenImporter → tokens.uss(UITK 변수) + DesignTokens.asset(uGUI ScriptableObject) 동시 자동생성. 드리프트 가드 = token_drift_check.py. 손작성 금지.

08

인터랙션 · HUD · 라이브옵스 원칙

L2 인터랙션

  • 하단·엄지 우선 — 주 액션 우하단, 상단 코너=재화/상태 전용
  • 제스처는 항상 명시적 버튼 대안 병행 · 하단탭 ≤5
  • FTUE: 첫 60초에 코어 루프 체험 · 가입/과금/광고는 뒤로

L3 인게임 HUD

  • 빈 화면에서 시작 — 상시 노출 정당화되는 것만
  • 위계: Critical(저체력·예고) > Important > Info > Ambient
  • 이펙트가 적 위협 예고를 가리면 안 됨 · 주스는 증폭기지 대체재 아님
  • 데미지 넘버: 크리만 강조·일반 누적·체력바 안 가림

L4 라이브옵스

  • 가치 투명성 · 가챠 확률+천장 노출 · 다크패턴 금지
  • 리텐션 3계층: 코어(30~90초) → 메타(3~5일) → 소셜
  • 처벌 아닌 보상 · 푸시는 가치제안(죄책감 금지)
