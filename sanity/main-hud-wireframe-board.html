스테이지 선택 → 편성 → 전투

"정적 존 배치"가 아니라 플레이어 여정(코어 루프)부터 설계한 가로(landscape) 와이어. 메인 = 스테이지 선택·진행 중심 / 편성 = 확인·조정(덱 유지) + 적 위협 응수 / 전투 HUD = 난전 가독.

갱신 2026-06-25 · 편성 공간 재배분(덱 우선 62%·2줄 로스터·전투시작 우하단·취소 인접) + 전투 HUD 일시정지/설정 생략 판단 · 매체 게이트 #8 = 레이아웃 = 캔버스 OK

↻ 전제 정정 — 왜 이전 보드를 버렸나

이전 보드는 메인 영웅 프레즌스 + 영웅 배치 A/B/C 가 전제였음. 폐기 이유 = ① 영웅 선택은 편성(스테이지 후)에서 일어남 → 메인 영웅 진열은 오해 유발 ② 이 장르(코스트 실시간 소환 레인배틀)의 주인공은 영웅 portrait 가 아니라 스테이지 진행 + 덱 ③ 덱은 적을 보고 짜는 게 전략적 정답(응수) → 메인 사전 편성은 "이른 결정". 새 전제: 메인 = 스테이지 선택 중심 · 편성 = 스테이지 맥락 안에서.

편성 설계 포인트 — 핵심 화면

  • 덱 영역 우선 (~62%) UX 판단 편성 = 주 인터랙션 → 덱에 공간 더·슬롯 크게. 적 진형 = 참조(우측 컴팩트).
  • 로스터 = 2줄 가로 스크롤 사용자 보유 37유닛 = 1줄이면 스크롤 과다 → 2줄로 한눈·스크롤 절반.
  • 전투 시작 = 우하단 사용자 오른손 엄지 · "덱 빌드(좌) → 적 확인(우) → 시작(우하)" 흐름.
  • 취소(뒤로) = 전투 시작 위 사용자 상단서 이동. ★ 미스탭 가드 = 취소 작게·점선(2차) / 전투 시작 크게·글로우(1차) 위계 분리.
  • 좌 덱 / 우 적 전투(아군 좌·적 우)와 일치. 적 = 진형 미리보기 + 위협 표기(상설·진형형·CEO Q5).
